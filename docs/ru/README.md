# Titan Core — Документация

> Переосмысление промышленной автоматизации для надёжного, эффективного и современного завода.

---

## Манифест

Промышленное ПО не менялось 30 лет.

Заводы по всему миру работают на системах которые были бы неприемлемы в любом другом домене. Windows XP в производственных системах в 2025 году — не редкость, это норма. Historian-базы которые не могут ответить на простой вопрос "что происходило на прошлой неделе" быстрее чем за несколько минут. Конфигурации без контроля версий. Интерфейсы из 1998 года. Счета на $200,000 за функциональность которую веб-разработчик воспроизведёт за месяц.

Мы принимаем это потому что "так устроено промышленное ПО."

**Titan Core — это отказ от этого принятия.**

Не очередная SCADA. Не улучшение существующих подходов. Это архитектурный чертёж того, каким должен быть современный завод — построенный на инструментах которые уже существуют и доказали свою надёжность в самых требовательных системах мира.

## Для кого этот проект

**АСУ ТП инженер и технолог**
Ты знаешь производство лучше любого программиста. Ты видел как системы падают в 3 ночи и знаешь почему нельзя просто перезапустить процесс. Titan Core даёт тебе современный инструментарий без необходимости становиться веб-разработчиком. Визуальный редактор, YAML конфиги, знакомая среда для ST программирования.

**Разработчик**
Ты знаешь Rust, Elixir, React. Titan Core открывает для тебя огромный рынок промышленной автоматизации через инструменты которые ты уже знаешь. Git, CLI, code review, TypeScript — всё как в обычной разработке. Без проприетарных сред и dongles.

**Системный интегратор**
Ты устал продавать Siemens за $300k когда задача решается за $30k. Titan Core даёт тебе референсную архитектуру, аргументы для разговора с заказчиком и маржу которой нет у конкурентов продающих лицензии.

**Визионер и евангелист**
Ты видишь что промышленный мир стоит перед переломом. NIS2 давит на безопасность OT-систем. Поколение КИПиА-инженеров уходит на пенсию. Заводы ищут способ не зависеть от одного вендора вечно. Ты хочешь быть частью этого перелома — и иметь архитектурный язык чтобы говорить об этом убедительно.

## Статус проекта

Titan Core — это концепция и архитектурная документация. Кода пока нет.

Это сделано намеренно. Прежде чем писать код — нужно выработать убеждение в правильности архитектуры. Код написанный без убеждения — технический долг с первого дня.

Если ты читаешь это и думаешь "это правильно" — открой Issue. Напиши что думаешь, где видишь проблемы, что упущено. Именно так строятся правильные вещи.
